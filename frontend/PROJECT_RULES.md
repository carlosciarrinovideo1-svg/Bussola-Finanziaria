# PROJECT RULES

## Regola 1
Meglio una casa piccola funzionante che un castello incompleto.

## Regola 2
Ogni Sprint termina con:
- Test
- Commit
- Push
- Repository pulito

## Regola 3
Mai utilizzare dati inventati come se fossero dati reali.

## Regola 4
La Dashboard non accede mai direttamente alle fonti ufficiali.

## Regola 5
Ogni modifica deve avere uno scopo preciso e verificabile.

## Regola 6
Prima di iniziare un nuovo Sprint il repository deve essere pulito.

## Regola 7

Il repository deve contenere tutte le informazioni necessarie per comprendere il progetto senza dipendere dalla memoria degli sviluppatori o dalle conversazioni.
